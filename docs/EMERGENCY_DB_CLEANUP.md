# Emergency Postgres Cleanup

Use this only when Render Postgres is full and the EasyMarket web service cannot query the database.

## Safe order

1. Stop/suspend the Render web service so it stops writing new rows.
2. Open the Render Postgres shell for `polik-db`.
3. Run the SQL below.
4. Start the web service again.

## What is kept

Do not truncate balances, ledgers, positions, trades, deposits, or withdrawals. They are the financial/audit state.

## Emergency SQL

```sql
-- Biggest disposable table: chart/feed ticks.
TRUNCATE TABLE price_ticks RESTART IDENTITY;

-- Disposable chat/comment history older than 3 days.
DELETE FROM market_comments
WHERE created_at < now() - interval '3 days';

-- Disposable scanner raw events after they are old.
DELETE FROM usdt_deposit_events
WHERE created_at < now() - interval '30 days';

-- Expired/cancelled deposit requests are not financial state.
DELETE FROM usdt_deposit_intents
WHERE status IN ('expired', 'cancelled', 'canceled')
  AND updated_at < now() - interval '30 days';

-- Old daily task claim rows can be regenerated by day rules.
DELETE FROM fire_task_claims
WHERE created_at < now() - interval '60 days';

-- Empty technical markets with no trades/positions/comments.
DELETE FROM markets m
WHERE m.status IN ('resolved', 'price_error', 'superseded')
  AND COALESCE(m.resolved_at, m.end_time, m.created_at) < now() - interval '14 days'
  AND NOT EXISTS (SELECT 1 FROM positions p WHERE p.market_id = m.id)
  AND NOT EXISTS (SELECT 1 FROM trades t WHERE t.market_id = m.id)
  AND NOT EXISTS (SELECT 1 FROM market_comments c WHERE c.market_id = m.id);

VACUUM (ANALYZE);
```

## After recovery

`PRICE_TICKS_DISABLED=false` is the default for live charts. Old raw ticks are pruned by the scheduled cleanup job; keep `DATABASE_CLEANUP_ENABLED=true` and a short `CLEANUP_PRICE_TICKS_HOURS` value on small Render Postgres plans. The app also has a startup safety cleanup that truncates only `price_ticks` when it grows above `CLEANUP_PRICE_TICKS_TRUNCATE_ABOVE_MB`.
